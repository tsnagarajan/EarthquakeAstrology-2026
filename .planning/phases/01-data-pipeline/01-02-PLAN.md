```python
import swisseph as swe

# MUST be called first — before any calculation
swe.set_ephe_path("/path/to/ephe/")  # or os.getenv("SE_EPHE_PATH")

# Julian Day from UTC date (use noon = 12.0 as hour to avoid day-boundary edge cases)
jd = swe.julday(year, month, day, 12.0)  # UT

# Planetary position — use calc_ut (UT input), NOT calc (TT input)
# Returns: (xx, ret) where xx = [longitude, latitude, distance, speed_long, speed_lat, speed_dist]
xx, ret = swe.calc_ut(jd, swe.SUN)
longitude = xx[0]       # 0–360 degrees
speed = xx[3]           # degrees/day; negative = retrograde
is_retrograde = speed < 0

# Planet constants (pysweph):
# swe.SUN, swe.MOON, swe.MERCURY, swe.VENUS, swe.MARS
# swe.JUPITER, swe.SATURN, swe.URANUS, swe.NEPTUNE, swe.PLUTO
# swe.CHIRON (asteroid 2060)
# swe.MEAN_APOG  — Lilith (Mean Black Moon)
# swe.MEAN_NODE  — North Node (Mean)

# Sidereal mode for Vedic nakshatras (Lahiri ayanamsha)
swe.set_sid_mode(swe.SIDM_LAHIRI)
xx_sid, ret_sid = swe.calc_ut(jd, swe.SUN, swe.FLG_SIDEREAL)
sidereal_lon = xx_sid[0]
nakshatra_num = int(sidereal_lon / (360 / 27))  # 0–26
# Reset to tropical after sidereal calc if mixing both in same session
swe.set_sid_mode(0)  # back to tropical

# Zodiac sign from tropical longitude
sign_num = int(longitude / 30)  # 0=Aries, 1=Taurus, ..., 11=Pisces
SIGN_NAMES = ['Aries','Taurus','Gemini','Cancer','Leo','Virgo',
              'Libra','Scorpio','Sagittarius','Capricorn','Aquarius','Pisces']

# Nakshatra names (27 nakshatras in Vedic system)

]
```

    **`compute_day(date_str)` function:**
    - Parses date_str as "YYYY-MM-DD"
    - Computes `jd = swe.julday(year, month, day, 12.0)` — ALWAYS 12.0 for UTC noon
    - For each planet in PLANETS:
      - Calls `swe.calc_ut(jd, planet_id)` — use calc_ut, never calc
      - Extracts longitude = xx[0], speed = xx[3]
      - sign_num = int(longitude / 30) % 12
      - retro = speed < 0
      - For nakshatra: call `swe.set_sid_mode(swe.SIDM_LAHIRI)`, then `swe.calc_ut(jd, planet_id, swe.FLG_SIDEREAL)` to get sidereal longitude, then nakshatra_num = int(sid_lon / (360/27)) % 27, then reset with `swe.set_sid_mode(0)` (back to tropical for next planet)
      - Stores {prefix}lon, {prefix}sign_num, {prefix}sign, {prefix}retro, {prefix}nakshatra_num, {prefix}nakshatra
    - Returns dict with 'date' key plus all planet columns

    **`compute_aspects(row)` function:**
    - Takes the row dict from compute_day
    - Iterates all pairs from PLANETS keys using itertools.combinations
    - For each pair (p1, p2) and each aspect (name, angle):
      - angular_diff = abs((row[f'{p1}_lon'] - row[f'{p2}_lon']) % 360)
      - If angular_diff > 180: angular_diff = 360 - angular_diff
      - is_aspect = abs(angular_diff - angle) <= ORBS
      - Column name: f"{p1}_{p2}_{name}" = 1 if is_aspect else 0
    - Returns dict of aspect columns

    **`compute_date_range(start_date, end_date)` function:**
    - Generates all dates in range (inclusive) using pd.date_range
    - Uses tqdm over dates
    - For each date: calls compute_day + compute_aspects, merges dicts
    - Returns list of row dicts

    **`main()` entry point:**
    - argparse: --start-date (default "1900-01-01"), --end-date (default "2026-12-31"), --output (default "data/raw/ephemeris.csv")
    - Calls setup_ephemeris()
    - Calls compute_date_range
    - Builds DataFrame, writes to CSV
    - Prints: "Computed {len(df)} days. Columns: {len(df.columns)}. Saved to {output}"

    **Guard:** `if __name__ == '__main__': main()`

    IMPORTANT: This script will take 5-30 minutes to run for the full 1900-2026 range. This is expected. Do not attempt to optimize with multiprocessing in this plan — correctness first.
