<objective>
Validate the ephemeris.csv output against JPL Horizons reference values for at least 10 spot-check dates, logging PASS/FAIL results to an audit file. This is a one-time accuracy gate — if it fails, the ephemeris script has a UTC or ephe_path bug that must be fixed before any downstream feature engineering begins.

Purpose: Ephemeris errors are the highest-recovery-cost bug in this project (PITFALLS.md rates recovery as HIGH cost). A validation script that runs in under 30 seconds catches degree-level errors before they propagate into 46,000 rows of features and a retrained model.
Output: pipeline/data/validate_ephemeris.py, data/validation/ephemeris_spot_check.log
</objective>

  <action>
    Create `pipeline/data/validate_ephemeris.py`:

    **Imports:** pandas, pathlib, logging, sys, os, datetime, dotenv

    **JPL_REFERENCE_VALUES list** (hardcoded):
    Use the exact values from the interfaces block above. These are tropical geocentric ecliptic longitudes pre-verified against JPL Horizons web UI. Add a comment noting: "Values verified against JPL Horizons (https://ssd.jpl.nasa.gov/horizons/) — geocentric, apparent, ecliptic longitude, J2000 frame. Tolerance is 0.5 deg; Swiss Ephemeris DE431 should agree to < 0.01 deg."

    **TOLERANCE = 0.5** (degrees)

    **`run_spot_checks(df)` function:**
    - Accepts DataFrame with 'date' column (string "YYYY-MM-DD") and planet longitude columns
    - Sets 'date' as index if not already
    - For each (date_str, planet_col, expected_lon) in JPL_REFERENCE_VALUES:
      - Look up row by date_str; if not found, append result with passed=False, reason="date not found"
      - Get computed = df.loc[date_str, planet_col]
      - delta = abs(computed - expected_lon)
      - Handle 360-degree wrap: if delta > 180, delta = 360 - delta
      - passed = delta <= TOLERANCE
      - Append dict: {date, planet: planet_col, computed, reference: expected_lon, delta, passed}
    - Returns list of result dicts

    **`format_log(results)` function:**
    - Returns formatted string with header, one line per result:
      `[PASS] 2000-01-01 sun_lon: computed=280.52 reference=280.5 delta=0.02`
      `[FAIL] 1900-01-01 sun_lon: computed=195.3 reference=280.5 delta=85.2`
    - Appends summary: "PASSED: 9/10, FAILED: 1/10"

    **`main()` entry point:**
    - argparse: --ephemeris (default "data/raw/ephemeris.csv"), --log (default "data/validation/ephemeris_spot_check.log")
    - Checks that ephemeris CSV exists; exits with code 2 and helpful message if not ("Run python pipeline/data/ephemeris.py first")
    - Reads ephemeris CSV
    - Calls run_spot_checks(df)
    - Writes log via logging.FileHandler to --log path; also prints to stdout
    - Creates data/validation/ directory if it does not exist
    - If any result has passed=False: print "VALIDATION FAILED — see {log_path}" and sys.exit(1)
    - If all pass: print "ALL SPOT CHECKS PASSED — {n} checks in {log_path}" and sys.exit(0)

    **Guard:** `if __name__ == '__main__': main()`

    Create `data/validation/.gitkeep` so the validation output directory is tracked in git (the .log file itself should be gitignored since it's generated).

    Add to .gitignore: `data/validation/*.log`
  </action>
